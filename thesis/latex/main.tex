% ABSTRACT
\chapter*{TÓM TẮT}
\addcontentsline{toc}{chapter}{TÓM TẮT}

Tấn công brute-force SSH là một trong những mối đe dọa phổ biến và dai dẳng nhất trong lĩnh vực an ninh mạng. Các công cụ phòng chống truyền thống như Fail2Ban hoạt động dựa trên ngưỡng tĩnh, chỉ phản ứng sau khi tấn công đã xảy ra và dễ bị lẩn tránh bởi các kỹ thuật tấn công tiên tiến như low-and-slow hoặc distributed brute-force.

Khóa luận này trình bày việc thiết kế, triển khai và đánh giá một hệ thống phát hiện và phòng chống tấn công brute-force SSH sử dụng trí tuệ nhân tạo, với khả năng dự đoán sớm (early prediction) các cuộc tấn công trước khi chúng leo thang. Hệ thống sử dụng thuật toán Isolation Forest làm mô hình phát hiện bất thường chính, kết hợp với cơ chế ngưỡng động EWMA-Adaptive Percentile (Exponentially Weighted Moving Average kết hợp Percentile thích ứng) để cung cấp khả năng cảnh báo sớm hai mức (EARLY\_WARNING và ALERT).

Bộ dữ liệu nghiên cứu gồm 174.250 dòng nhật ký SSH, bao gồm dữ liệu tấn công thực tế từ hệ thống honeypot (119.729 dòng, 679 địa chỉ IP tấn công) và dữ liệu hành vi bình thường từ môi trường mô phỏng (54.521 dòng, 64 người dùng). Hệ thống trích xuất 14 đặc trưng hành vi theo cửa sổ thời gian 5 phút cho mỗi địa chỉ IP nguồn.

Kết quả thực nghiệm cho thấy Isolation Forest đạt F1-Score 93,74\%, Accuracy 90,31\% và False Positive Rate 29,00\% sau tối ưu hóa, cải thiện đáng kể so với baseline (F1 tăng 5,11\%, FPR giảm 47,92\%). So sánh với hai mô hình đối sánh LOF (F1=89,94\%) và One-Class SVM (F1=94,55\%), Isolation Forest được chọn làm mô hình chính nhờ khả năng tạo anomaly score liên tục phù hợp cho thuật toán ngưỡng động.

Hệ thống được triển khai hoàn chỉnh với kiến trúc microservices, bao gồm: pipeline phát hiện thời gian thực (Python asyncio), backend API (FastAPI), dashboard giám sát (React), trực quan hóa (ELK Stack/Kibana), cảnh báo (email/WebSocket), và tự động chặn IP (Fail2Ban). Toàn bộ hệ thống được đóng gói bằng Docker Compose (9 services) để đảm bảo khả năng triển khai trong môi trường production.

\vspace{0.5cm}
\noindent\textbf{Từ khóa:} SSH, brute-force, phát hiện bất thường, Isolation Forest, ngưỡng động, EWMA, dự đoán sớm, học máy, ELK Stack, Fail2Ban.

\chapter*{ABSTRACT}
\addcontentsline{toc}{chapter}{ABSTRACT}

SSH brute-force attacks remain one of the most prevalent and persistent threats in cybersecurity. Traditional defense tools such as Fail2Ban operate on static thresholds, reacting only after an attack has occurred and being easily evaded by advanced techniques such as low-and-slow or distributed brute-force attacks.

This thesis presents the design, implementation, and evaluation of an AI-powered system for detecting and preventing SSH brute-force attacks with early prediction capability. The system employs Isolation Forest as the primary anomaly detection model, combined with an EWMA-Adaptive Percentile dynamic threshold mechanism to provide two-level early warnings (EARLY\_WARNING and ALERT).

The research dataset comprises 174,250 SSH log entries, including real attack data from a honeypot system (119,729 lines, 679 attacking IPs) and normal behavior data from a simulated environment (54,521 lines, 64 users). The system extracts 14 behavioral features per 5-minute time window for each source IP.

Experimental results demonstrate that Isolation Forest achieves an F1-Score of 93.74\%, Accuracy of 90.31\%, and False Positive Rate of 29.00\% after optimization. The system was benchmarked against LOF (F1=89.94\%) and One-Class SVM (F1=94.55\%). The complete system is deployed as a Docker Compose stack with 9 services, including real-time detection pipeline, FastAPI backend, React dashboard, ELK Stack visualization, email/WebSocket alerts, and Fail2Ban integration.

\vspace{0.5cm}
\noindent\textbf{Keywords:} SSH, brute-force, anomaly detection, Isolation Forest, dynamic threshold, EWMA, early prediction, machine learning, ELK Stack, Fail2Ban.

\section{Dynamic Threshold}
\begin{lstlisting}[language=Python, caption={EWMA-Adaptive Percentile Algorithm}]
def evaluate(self, anomaly_score, timestamp=None, source_ip=None):
    # Step 1: Update EWMA
    if self._ewma is None:
        self._ewma = anomaly_score
    else:
        self._ewma = self.alpha * anomaly_score + (1-self.alpha) * self._ewma

    # Step 2: Adaptive threshold
    self._score_buffer.append(anomaly_score)
    adaptive_threshold = np.percentile(list(self._score_buffer),
                                        self.base_percentile)

    # Step 3: Two-level decision
    early_warning_threshold = adaptive_threshold / self.sensitivity_factor
    if self._ewma >= adaptive_threshold:
        return ThreatLevel.ALERT
    elif self._ewma >= early_warning_threshold:
        return ThreatLevel.EARLY_WARNING
    else:
        return ThreatLevel.NORMAL
\end{lstlisting}
